# h0 平台规范

## 接口规范

```
{API_HOST}/{服务}/{租户}/{路由}
```

- 使用平台多语言时，租户默认为当前 token 中的租户，未指定时默认 0
- 接口权限必须是平台层

## 前端路由规范

路由配置在 `packages/{package}/config/config.ts`：

```javascript
import { extendParentConfig } from '@hzerojs/plugin-micro';

const config = extendParentConfig({
  routes: [
    {
      path: '/hskp-platform/domain-manage',
      component: '@/pages/DomainManagement',
      title: 'hskp.platform.domainManagement.view.title',
    },
  ]
});

export default config;
```

- `@` 别名表示路径为 `{package}/pages`
- 每个包不一定有 routes，说明这个包都是被别人引用的
- 路由最外层页面必须使用 `formatterCollections` 加载多语言
- 常用路由属性：
  - `title`: 页面标题，使用多语言 key
  - `authorized`: `true`/`false`，是否需要鉴权
  - `layout`: `false` 时隐藏侧边栏和顶栏
  - `type`: `'vue'` 时使用 Vue 组件

## 多语言自动加载

`hzero.common` 在基于 h0 的项目中会被**所有页面自动请求**，不需要在 `formatterCollections` 中显式声明。项目自定义的 common promptKey（如 `hskp.common`）**不是**自动加载的，需要在 `formatterCollections` 的 `code` 数组中指定。

## formatterCollections 用法

`formatterCollections` 是一个 HOC（高阶组件），用于包裹页面组件，自动加载指定命名空间的多语言数据。

```javascript
import React from 'react';
import formatterCollections from 'utils/intl/formatterCollections';
import Loading from 'components/skeleton-loading/loading';

// 页面组件
const DomainManagement = () => {
  // ...
};

// HOC 包裹导出
export default formatterCollections({
  code: ['hskp.common', 'hskp.platform'],  // hskp.common 需要声明，只有 hzero.common 自动加载
  loadingComponent: () => <Loading />,
})(DomainManagement);
```

- `code` 数组指定需要加载的多语言命名空间
- `loadingComponent` 在多语言数据加载时显示，通常使用 `components/skeleton-loading/loading`
- **只有 `hzero.common` 是自动加载的**，项目自定义的 common promptKey 需要显式声明
- 每个使用 `formatterCollections` 的页面都必须引入 `Loading` 组件
